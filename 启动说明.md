# Type Editor - 一键启动说明

## 🚀 快速开始

### Windows 用户（推荐）

**最简单的方式：双击运行**
```
双击 start.bat 文件即可
```

或者使用 PowerShell：
```powershell
.\start.ps1
```

### Linux/macOS 用户

```bash
chmod +x start.sh
./start.sh
```

### 跨平台方式（所有系统）

```bash
npm start
```

## 📋 启动脚本说明

项目提供了多种启动方式，选择最适合你的：

### 1. start.bat (Windows 批处理)
- ✅ 最简单，双击即可运行
- ✅ 自动检查并安装依赖
- ✅ 在新窗口中启动前后端
- ⚠️ 仅支持 Windows

### 2. start.ps1 (PowerShell)
- ✅ 功能更强大
- ✅ 彩色输出，更美观
- ✅ 自动检查并安装依赖
- ⚠️ 需要 PowerShell（Windows 10+ 自带）

### 3. start.sh (Shell 脚本)
- ✅ Linux/macOS 专用
- ✅ 自动检查并安装依赖
- ✅ 后台运行，支持 Ctrl+C 停止

### 4. start.js (Node.js 跨平台)
- ✅ 支持所有操作系统
- ✅ 统一管理前后端进程
- ✅ 自动检查并安装依赖
- ✅ 支持优雅关闭（Ctrl+C）

## 🎯 使用场景

### 首次使用
脚本会自动检查并安装依赖，无需手动操作。

### 日常开发
直接运行启动脚本，前后端会自动启动。

### 停止服务

**Windows (批处理/PowerShell)**
- 关闭对应的命令行窗口即可

**Linux/macOS (Shell)**
- 按 `Ctrl+C` 停止所有服务

**Node.js 脚本**
- 按 `Ctrl+C` 优雅停止所有服务

## ⚙️ 环境要求

- Node.js 14+ （推荐 16+）
- npm 或 yarn

## 🔧 故障排除

### 问题1：端口被占用
```
错误: listen EADDRINUSE: address already in use :::3001
```
**解决方案**：
1. 关闭占用端口的程序
2. 或修改 `.env` 文件中的 `PORT` 配置

### 问题2：依赖安装失败
**解决方案**：
1. 检查网络连接
2. 清除 npm 缓存：`npm cache clean --force`
3. 删除 `node_modules` 后重新安装

### 问题3：PowerShell 执行策略限制
```
无法加载文件，因为在此系统上禁止运行脚本
```
**解决方案**：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 问题4：权限不足（Linux/macOS）
**解决方案**：
```bash
chmod +x start.sh
```

## 📝 注意事项

1. **首次启动**：会自动安装依赖，可能需要几分钟
2. **端口占用**：确保 3001 和 5000 端口未被占用
3. **数据库**：首次运行会自动创建数据库文件
4. **环境变量**：可以创建 `.env` 文件自定义配置

## 🎉 启动成功后

访问以下地址：
- 🌐 前端界面：http://localhost:5000
- 🔌 后端 API：http://localhost:3001

开始使用 Type Editor 吧！
